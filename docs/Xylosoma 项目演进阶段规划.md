# Xylosoma 项目演进阶段规划

> **文档定位**：在《技术白皮书（现实可操作版）》的全局愿景与伦理边界之下，定义**工程与产品可交付阶段**；与《架构设计 v0.1》中的 **Stage 1「静默的感知者」** 直接对应。  
> **关联文档**：[技术白皮书（现实可操作版）](./Xylosoma%20技术白皮书（现实可操作版）.md) · [架构设计 v0.1](./Xylosoma%20架构设计%20v0.1.md) · [阶段技术实现方案](./Xylosoma%20阶段技术实现方案.md) · [Stage 1 技术实现与落地规划](./Xylosoma%20Stage1%20技术实现与落地规划.md)

## 1. 规划原则

| 原则 | 说明 |
|------|------|
| **白皮书锚定** | 阶段划分不违背五项边界（无 PII、无神经数据、无个体操控、数据主权、代际公平）。 |
| **可验证交付** | 每一阶段有明确「能演示 / 能测量 / 能审计」的产出，避免长期悬空能力。 |
| **阶段可裁剪** | 资源有限时，可在单阶段内缩小地理范围或数据源数量，但**不放宽伦理与安全底线**。 |
| **与白皮书路线图对齐** | 下表中的「白皮书阶段」对应白皮书第八章时间表，便于对外叙述一致。 |

## 2. 阶段总览

| 工程阶段 | 代号 | 时间建议（可与白皮书交叉） | 一句话目标 | 白皮书对应 |
|----------|------|---------------------------|------------|------------|
| **阶段 0** | 预研与共识 | 2026 H1 | 数据源与法务清单、威胁模型、Stage 1 范围冻结 | 概念验证前奏 |
| **阶段 1** | 静默感知（Stage 1） | 2026–2027 | 纯前端全球/区域**只读**可视与叙事，零自有后端、零用户画像 | 概念验证（展示与教育子集） |
| **阶段 2** | 节律与指标层 | 2027–2028 | 在受控环境中对**已脱敏/已聚合**数据进行节律与异常指标计算，可人工复核 | 概念验证（算法与闭环预案） |
| **阶段 3** | 区域聚合与治理 | 2028–2030 | 1–3 个试点区域的数据协议、退出/静默机制、社区仪表板 | 区域扩展 |
| **阶段 4** | 耦合模型与试验性响应 | 2030+ | GNN/物理耦合等**研发级**模型；响应限于**授权下的环境级**试点 | 区域扩展 → 系统集成过渡 |
| **阶段 5** | 系统集成与规模化 | 2031+ | 市政/行业 API 在**合同与审计**前提下对接；跨部门编排 | 系统集成 |
| **阶段 6** | 全球网络与协同治理 | 2036+ | 跨国机制与生态信用等（以政策与合约为牵引） | 全球网络 |

说明：**阶段 1** 即为当前《架构设计 v0.1》的实现焦点；**阶段 2 起**必然引入后端、批处理或专有网络，与「纯静态站点」解耦，需在独立架构修订中展开。

## 3. 各阶段交付物（摘要）

### 阶段 0：预研与共识

- 公开数据源 **ToS / 速率 / CORS** 实测矩阵；不可用源的替代方案。
- **隐私与安全**：浏览器直连接口下的「第三方可见性」说明（IP、Referer 等）。
- **范围说明书**：Stage 1 图层列表、刷新策略、失败降级与性能预算。
- 与白皮书一致的**对外话术**：Stage 1 **不等于** 已实现「节律引擎 / 耦合模型 / 市政闭环」。

### 阶段 1：静默感知（Stage 1）

- 可部署的 **Vue 3 + Vite + Cesium** 单页应用；静态托管（如 Cloudflare Pages）。
- **api 白名单**、分组 Entity 更新、无自有数据库、无登录。
- 可重复构建的开源仓库与 **README 部署说明**。

### 阶段 2：节律与指标层

- 数据接入规范（对象存储 / 表 / 流）与 **≥1km² 或等效聚合** 的硬性校验规则。
- 节律提取与异常检测的**批处理或受控实时作业**；模型与数据版本可追溯。
- **人工可读**指标与「生态语言映射」原型（非自动干预）。

### 阶段 3：区域聚合与治理

- 社区数据授权、**节律静默**与数据剔除流程（含模型重训或特征开关策略）。
- 区域运营仪表板（可仍为只读 + 导出报告）；审计日志留存策略符合本地法。

### 阶段 4–6：略（随白皮书演进滚动细化）

- 每进入新阶段前，单独增加《架构设计 v0.x》修订，**禁止**用 Stage 1 技术栈冒充闭环能力。

## 4. 风险与依赖（跨阶段）

| 类型 | 说明 |
|------|------|
| **数据依赖** | 电网、交通、信令等通常需协议，不可写入 Stage 1 的「默认可用」假设。 |
| **合规** | 多国并行时按最严区域原则设计聚合与存储。 |
| **叙事风险** | 对外需区分「哲学愿景」「研究路线」「当前 Stage 可演示能力」。 |
