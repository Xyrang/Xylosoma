# Xylosoma 阶段技术实现方案

> **文档定位**：对应《项目演进阶段规划》中各阶段，给出**技术形态、推荐栈、边界与接口**，便于研发分步实施。  
> **关联文档**：[项目演进阶段规划](./Xylosoma%20项目演进阶段规划.md) · [Stage 1 技术实现与落地规划](./Xylosoma%20Stage1%20技术实现与落地规划.md) · [架构设计 v0.1](./Xylosoma%20架构设计%20v0.1.md)

---

## 1. 总体技术原则（全阶段）

1. **伦理内置**：默认最小化采集；聚合与访问控制设计优先于算法复杂度。  
2. **可观测**：关键作业有日志与指标（Stage 1 仅限前端错误与性能；自 Phase 2 起含流水线血缘）。  
3. **开放优先**：模型与**非敏感**工具链鼓励开源；密钥与合同数据不进公开仓库。  
4. **阶段递进**：后一阶段**可包裹或替换**前一阶段，避免「推倒重来」式架构赌博。

## 2. 阶段 0：预研与共识

| 维度 | 方案 |
|------|------|
| **产出形态** | 文档、表格、PoC 脚本（可选）；无要求上线产品。 |
| **工具** | 电子表格或轻量脚本（如 Python/notebook）做数据源探测；不谈统一技术栈。 |
| **安全** | 定义 **API 白名单域**、CSP 策略草案、第三方请求隐私说明。 |

## 3. 阶段 1：静默感知（Stage 1）

与《架构设计 v0.1》一致：**类 JAMstack / 纯静态前端**，**无自有服务器、无自有数据库、无用户系统**。

| 层级 | 实现要点 |
|------|----------|
| **客户端** | Vue 3 + Vite + Pinia；CesiumJS；可选 D3 / ECharts 做二维附图。 |
| **数据** | 浏览器 **fetch** 直连**公开或半公开** HTTPS API；`api.js` **域名白名单**；禁止随意引入统计分析脚本。 |
| **托管** | 静态资源托管（如 Cloudflare Pages / 同类）；全站 HTTPS。 |
| **地图与图层** | 免 Ion 额度的影像/地形方案需 **按版本实测**（ArcGIS/OSM/GIBS 等 ToS 与可用性）；时间动态图层需绑定时钟与**最近可用产品时间**。 |
| **性能** | Entity **分集合管理**（避免 `removeAll` 误删）；移动端 LOD、抽稀、限制同时渲染数量。 |
| **隐私表述** | 「本站不记录用户、不设 first-party 追踪」；同时披露用户设备仍会访问第三方 API。 |

**明确不做**：用户账号、服务端持久化、个人行为画像、闭环节律模型、市政控制指令下发。

---

## 4. 阶段 2：节律与指标层

引入**受控计算平面**（与公开静态站点分离）。

| 层级 | 实现要点 |
|------|----------|
| **数据入湖/仓** | 对象存储 + 分区表（按区域、时间）；或托管数据湖服务；**仅接收已审批管道**数据。 |
| **计算** | 批处理（如每日/每小时）为主；流式为辅；调度器（如开源工作流或云厂商编排）。 |
| **算法** | 白皮书中的节律分解、异常检测等在此阶段**小范围验证**；输出为指标序列与报告，不自动执行物理世界动作。 |
| **服务接口** | 可选 **BFF / 边缘函数** 对外提供**聚合指标** JSON（仍无个体粒度）。 |
| **前端** | Stage 1 应用可扩展为「指标层只读消费者」，或通过新路由嵌入图表；认证视公开程度可为匿名限流或合作方密钥。 |

---

## 5. 阶段 3：区域聚合与治理

| 层级 | 实现要点 |
|------|----------|
| **多租户 / 社区** | 逻辑隔离（按区域/社区 ID）；数据与模型配置**分区**；**节律静默** = 停止摄取 + 特征剔除 + 版本标记。 |
| **合规模块** | 同意书与授权记录存储（加密、留存期限可配置）；审计日志不可篡改或定期导出见证。 |
| **产品** | 社区/城市仪表板；导出 PDF 或签名报告；可能仍 **不** 连接市政执行器。 |

---

## 6. 阶段 4：耦合模型与试验性响应

| 层级 | 实现要点 |
|------|----------|
| **模型** | 异构图 + GNN 或混合物理仿真；训练与推理环境与生产数据隔离；**模型卡**与复现脚本。 |
| **仿真** | 长期生态影响可对接学术或伙伴的高性能仿真（非 Stage 1 浏览器职责）。 |
| **响应** | **建议 / 审批 / 人工触发** 链；任何自动化须**合同授权、范围封顶、急停**。 |

---

## 7. 阶段 5：系统集成

| 层级 | 实现要点 |
|------|----------|
| **集成** | 市政 IoT、调度、保险等 **API 适配器**；每适配器独立版本与熔断。 |
| **身份与权限** | 机构级 IAM；与公民侧匿名访问严格分离。 |
| **SLO** | 可用性、延迟、RTO/RPO 按系统等级定义。 |

---

## 8. 阶段 6：全球网络

以**治理与标准**为主导：跨境数据合同、生态信用接口规范、联盟运营流程；技术上演进为联邦或分层 hub，**不在本文细化**。

---

## 9. 跨阶段接口预留（非强制实现）

| 接口概念 | 用途 |
|----------|------|
| **指标目录（Catalog）** | Stage 2+ 统一描述指标名称、单位、空间分辨率、延迟。 |
| **区域策略包** | Stage 3+ 描述该区域数据源列表、静默状态、展示开关。 |
| **响应意图（Intent）** | Stage 4+ 仅描述「建议的环境动作类型」编码，**不等于**直控设备。 |
