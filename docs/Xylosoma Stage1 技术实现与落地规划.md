## 2. 技术架构（冻结项）

| 组件 | 选型 | 备注 |
|------|------|------|
| 框架 | Vue 3 + Vite | 与架构 v0.1 一致 |
| 状态 | Pinia | 时间轴播放、图层开关等全局状态 |
| 地球引擎 | CesiumJS | `vite-plugin-cesium` 打包；**Ion Token 是否使用**以满足地形/影像为准，优先评估免配额方案与 ToS |
| 请求 | **原生 fetch** | 统一封装：超时、重试、错误分类；**禁止**未审查的第三方追踪脚本 |
| 样式 | Tailwind CSS（可选）+ 全局暗夜主题 | 与「暗夜与脉搏」视觉一致 |
| 托管 | Cloudflare Pages（或等价静态托管） | 构建命令 `npm run build`，输出 `dist` |

**隐私与安全（实现对用户的承诺）**

- 无登录、无自有后端、无 first-party Cookie 画像。  
- **须在产品文案中说明**：页面加载后，浏览器仍向 USGS、NASA、OpenAQ、底图服务商等**发起网络请求**，对方可能按常规记录了访问者的 IP 等网络元数据（与「本站不追踪」并行说明，避免过度承诺）。  
- Content-Security-Policy：默认仅允许白名单域脚本与连接；随 `api.js` 迭代更新。

## 4. 数据层实现要点

### 4.1 白名单配置（`config/api.js`）

- 枚举：名称、`baseURL`、路径模板、**最小刷新间隔**、失败退避（避免对公共 API 洪泛）。  
- 构建时或运行时不加载「清单外」地址。

### 4.2 图层更新规范

| 反模式 | 推荐做法 |
|--------|----------|
| `viewer.entities.removeAll()` 刷新单一数据源 | 为地震、AQI、野火等使用**独立集**或带前缀的 id，仅更新对应集合 |
| 写死单层 WMS/WMTS 的固定日期 | 对接 GIBS 等产品时解析 **当前或最近可用时间**；缺图时降级为仅底图 |
| 假设示例 URL 可直接用于 FIRMS | 以 NASA FIRMS **官方文档**为准配置 API Key 与路径；无 Key 则 Stage 1 **可暂不提供**野火层 |

### 4.3 Cesium / 遥测

- 查阅**所用 Cesium 版本**官方说明：是否存在 `config.analytics` 或等价开关；**以实测构建为准**。  
- 不依赖未经确认的「一键关遥测」代码片段。

### 4.4 初始数据源建议（Stage 1 最小集）

可在「第一次上线」时只启用已通过 PoC 的层，其余标为「实验」开关默认关闭：

| 优先级 | 数据 | 用途 |
|--------|------|------|
| P0 | USGS GeoJSON（地震） | 核心叙事「脉搏」 |
| P0 | 暗色底图（已测 ToS 之一） | 视觉基调 |
| P1 | Open-Meteo 等（无密钥风/温） | 大气叙事，注意 CORS |
| P1 | OpenAQ 最新（限流） | 城市呼吸，注意请求频率 |
| P2 | NASA GIBS 动态云/夜光 | 须在时间维与模板 URL 上单独联调 |
| P3 | FIRMS 等 | 密钥与条款就绪后再开 |

## 5. 性能与兼容性

| 项 | 指标 / 做法 |
|----|-------------|
| 移动端 | 中低端机：限制同时实体数、降低粒子与圆柱密度；可选「轻量模式」仅地震 + 底图 |
| 内存 | 定时清理失效 primitive；避免重复堆叠未释放的 ImageryLayer |
| 刷新 | 全局调度「错峰」请求，合并同域请求间隔 |
| 容错 | API 失败保留上一帧；UI 角标提示「数据源暂不可用」 |

## 6. 落地规划（四周节奏，可按团队调整）

| 周次 | 目标 | 验收 |
|------|------|------|
| **第 1 周** | 仓库初始化；Cesium + 暗夜底图；Viewer 控件裁剪 | 本地与预览环境可稳定旋转地球 |
| **第 2 周** | P0 地震层；`collections` 分组更新；`fetchJson` 封装 | 刷新不闪删其他层；错误不白屏 |
| **第 3 周** | 增选 1–2 个 P1 层（通过 PoC）；基础 CSP；README 部署步骤 | 静态构建可复现；文档与代码一致 |
| **第 4 周** | 视觉与文案（哲学化描述）；移动端抽样测试；**生产部署** | Lighthouse/真机可接受；隐私文案已审 |

**上线检查清单（节选）**

- [ ] `api.js` 与 CSP connect-src 一致  
- [ ] 无控制台未处理致命错误  
- [ ] 无私钥进仓库；若未来有 Key，走托管环境变量（Stage 1 尽量零密钥）  
- [ ] README 标明：本阶段能力边界与白皮书关系
